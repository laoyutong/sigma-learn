# Learner Profile
Updated: 2026-03-23

## Learning Style
- Preferred explanation mode: 能够快速理解并运用隐喻（如"智能管家"），对具体场景（如咖啡店、股票、订票、游戏开发）的分析能力很强。在代码逻辑上，对伪代码和流程图的接受度极高。偏好每个概念配套 **完整的 Markdown 总结文件**（visuals/*.md），不需要 HTML 可视化。内容要完整详尽，不精简——包含完整代码示例、对比说明、ASCII 流程图、选择指南等。
- Code Language Preference: TypeScript（明确要求，非 Python）
- Pace: fast。能够迅速抓住核心概念，不需要过多的重复解释。
- Responds best to: 场景模拟题、角色扮演题、需要自己拆解步骤的推理题、代码逻辑推演题。
- Struggles with: 无明显短板。

## Misconception Patterns
- 容易将带有 UI 的产品体验（如 ChatGPT 会话记忆）代入到底层 API 的工作原理中。但只要给出极端场景（如高并发）的反例，就能立刻顿悟。
